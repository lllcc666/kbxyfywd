# 任务自动化系列分批记录（第 1 批）

## 1. 分批原则

这一批只记录当前已经能从 `data/taskinfo.xml`、`TaskInfoXMLParser`、`TaskXMLParser` 和现有 AS3/Hook 里直接确认的内容。

- 只按“一个系列块”记录
- 只写已验证的 `taskID` / `subtaskID` / `after` / `before` / `targets`
- 发现主题名和任务根名不一致时，两个都要记
- 没有证据的部分统一标记为 `待确认`

## 2. 本批分析流程

这一批不是直接“抄任务名”，而是按同一套流程拆出来的：

1. 先在 `data/taskinfo.xml` 里定位目标 `taskID`
2. 再读它的 `subtaskID` 链和 `describe`，确认它是单步、短链还是长链
3. 再看 `targets` 和 `after`，判断它是不是主线、支线、剧情块或主题占位
4. 再回到 `TaskArchivesVersion3` 和相关 AS3，确认它在 UI 里挂在哪个主题下
5. 再根据 `TaskXMLParser` 判断是否存在对话、选择、战斗、场景切换、动画收口
6. 如果出现明显的特殊 packet 或特殊恢复点，单独拆成一条规则

原则上，这一批只做三件事：

- 确认任务块边界
- 标清执行顺序
- 标记待确认部分

## 3. 魔晶系

这里的“魔晶系”指的是 `TaskArchivesVersion3` 里归到 `魔晶战记` 主题下的相关链，以及 `taskinfo.xml` 中围绕魔晶石、魔晶能量、魔晶罗盘展开的任务块。

### 已验证

- `2002000 山崩地裂`，起点是 `2002001 可怕的石头`、`2002002 魔晶射线`、`2002003 失落的经文`
- `2009000 混世魔童`，已确认子任务包含 `2009001`、`2009002 魔童现世`、`2009003`、`2009004`、`2009005 菩萨的真相`
- `2014000 魂石传说（上）`，已确认子任务包含 `2014001`、`2014002 紫竹林解谜`、`2014003`、`2014004`
- `2015000 魂石传说（下）`，已确认子任务包含 `2015001` 到 `2015007`
- `3072000 暗灭星陨`，属于魔晶能量相关的后续任务块
- `7004000 魔晶罗盘`，属于魔晶能量衍生的单步任务块

### 分析路径

- 先从 `2002000` 读出“魔晶石发现”的起点，再看它和后续魔晶任务的描述是否共享同一剧情线
- `2009000` 这一块继续围绕魔晶石、伪魔晶石和不同场景展开，说明它不是单纯的对话任务
- `2014000` 和 `2015000` 是典型的两段式剧情，前段铺垫，后段收束，必须按上下章看
- `7004000` 只有一个子任务，适合当作单步收口型任务记录，不要强行合并进长链

### 执行约束

- 这类任务不要按“看起来像魔晶”的描述合并
- 必须先看 `taskID` 是否连续，再看 `subtaskID` 是否能形成实际链
- 只要出现跨场景、道具交付、战斗、收口动画，就不能用同一套固定顺序硬推
- `TaskXMLParser` 的 `choose flag="3"`、`battle`、`flash` 节点必须单独处理

### 当前判断

魔晶系不是一条单任务链，而是多个任务块串起来的剧情线。后续如果继续补这一系，建议按 `taskID` 分块输出，而不是试图一次性写完所有魔晶相关任务。

## 4. 地心系

当前快照里，能直接确认的代表块是：

- `3236000 地心来客`
- 子任务只有 `3236001`
- `after="2235"`

### 当前状态

这条链在当前仓库快照里呈现为短链，且 `taskinfo.xml` 里当前只暴露出一个可直接确认的子任务段。

### 分析路径

- 先确认它的根任务是 `3236000 地心来客`
- 再读出它当前只有一个子任务 `3236001`
- 再看描述里的“研究院 / 叮叮博士”是不是和后续其他任务块共享同一入口
- 因为当前只看到单步块，所以先把它当成剧情引子，不要提前补成完整系列

### 编写规则

- 不要直接把“地心传说”写成已确认任务名，当前快照没有这个精确字符串
- 先以 `地心来客` 作为已验证根块
- 如果后续拿到新的 `tasktheme.xml` 或更多 `taskinfo.xml` 片段，再补主题映射

## 5. 卡布系

当前快照里能直接确认的卡布相关块有：

- `2016000 紫月赤灵`
- `3256000 代号：卡布行动！`
- `7011000 真假小卡布`

### 已验证

- `2016000` 不是单步任务，当前至少能看到 `2016001 开启卡布特训`、`2016002 智慧的考验`、`2016003 勇气的考验`、`2016004 获得紫月赤灵`
- `3256000` 当前是短链，子任务 `3256001` 直接指向研究院
- `7011000` 当前也是单步块，围绕档案室和贝拉展开

### 分析路径

- `2016000` 要按“先开场、再双考验、最后领奖”的四段式去理解，不能当作一条简单直线
- `3256000` 更像剧情接力点，分析时重点看它和前后任务块的衔接，而不是它自身有多少子任务
- `7011000` 是独立短链，适合作为“UI 主题块里的单任务样例”
- 这批里没有直接出现“卡布历险”这个精确字面，因此只能把它归为更高层的待确认主题

### 当前状态

- 当前快照没有直接出现“卡布历险”这四个字
- 先把它当成更高层的主题占位，不要把它硬写成已验证根名
- 这类任务块要继续按“剧情块”拆，不要按口头印象合并

## 6. 下一批怎么补

下一批只补一个系列方向，建议顺序如下：

1. 先补完整一个 `taskID` 块
2. 再补它的前置和后续块
3. 再补 packet 顺序和回包点
4. 最后再补 UI/自动化上的收口规则

## 7. 统一模板

以后每个系列块都按这个模板写：

- 系列名
- 已确认根任务
- 子任务顺序
- 关键场景
- 关键 packet
- 中断点
- 恢复点
- 待确认项
